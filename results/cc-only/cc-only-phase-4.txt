  Phase 4 architecture sketch

  1. Write serializer (§11.1) — new tiny module src/writeLock.ts exporting
  withWriteLock(fn) that chains every write through a single promise. Reads stay
   un-synchronized. Every Phase-4 route handler runs its body inside
  withWriteLock. Cheap, single-process; matches §11.1's "single global write
  lock or equivalent."
  2. Validating mutation layer — a new src/mutations.ts (or split per-op) sits
  between the routes and the fixture primitives. Pattern for every write:

  1. Look up resources (404 if missing).
  2. Authorize: ownership / parent-ownership (403).
  3. Validate inputs syntactically (400) — name format, registered new owner.
  4. Apply tentatively via fixture primitives.
  5. evaluateAll(state). If violations → roll back and 409 with reason.
  6. Commit (do nothing — already applied) and return.

  2. Rollback is straightforward because the fixture primitives are reversible —
   the writer remembers the inverse step. For the multi-step delete, I'll
  compute the deletion region + detach set up-front (read-only walk), check
  preconditions, then apply unconditionally (no invariant re-check needed for
  delete — deletion strictly removes structure and can't create cycles,
  duplicates, or clashes).
  3. Routes — new src/routes/writes.ts mounted in app.ts:
    - POST /taxa (§6.1)
    - PATCH /taxa/{id} (§6.2) — accepts { name?, owner? }, validates name +
  invariant 3 across every containing tree (the invariant module gets this for
  free since evaluateAll iterates every root), validates owner is registered.
    - DELETE /taxa/{id} (§6.3)
    - PUT /taxa/{parentId}/children/{childId} (§6.4)
    - DELETE /taxa/{parentId}/children/{childId} (§6.4)

  All behind identityWithRegistry + requireWriter.
  4. Delete region computation (§6.3, the subtlest piece) — walk downward from
  N. At each step, include the current node if it's owned by U; if it's owned by
   someone else (and isn't N), don't include it and instead record
  (parentInRegion → otherOwnedChild) as a halt-frontier edge to detach.
  Recursion stops at other-owned children — even if a U-owned taxon sits below
  them, it's stranded (§14 cascade-deletion-never-removes-others bullet). Then
  the two preconditions:
    - P1: every region taxon (including N) is not shared — i.e.
  treesContaining(t).length ≤ 1 for each. (The §6.3 parenthetical notes
  halt-frontier taxa may be shared — they're not in the region.)
    - P2: N has zero parents, or exactly one parent owned by U.
  On commit: remove every region taxon from taxa and from every parent's
  childIds, and remove the halt-frontier edges.
  5. Test plans — test-plans/phase-4-initial.md (frozen at first commit), then
  phase-4-final.md at the end with a changelog. Initial plan written from spec
  before coding.
